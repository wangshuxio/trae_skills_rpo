# PRD Writer

## Description
帮助 AI 产品经理快速为 vibe coding 项目撰写一页纸 PRD。将模糊的"我想做一个XX"想法，梳理成可以直接喂给 AI 执行的产品需求文档。

## Triggers
- 帮我写个PRD
- 我想做一个XX
- 帮我梳理需求
- 我要vibe code一个产品

## Commands

### /write-prd
启动 PRD 撰写流程。

---

## 工作流程

### 第一步：判断用户状态

用户可能有两种情况：

**A. 想法很模糊**
只说一句"我想做一个XX"，没有更多背景。
→ 告知用户："好的，我们一步步来。我会问你几个问题，帮你把想法理清楚。"

**B. 需求已经比较清晰**
用户已经想好了要做什么，甚至已经列了功能清单。
→ 告知用户："看来你已经想得差不多了！我快速确认几个关键点就出PRD。"

---

### 第二步：需求澄清

用5个必问问题锁定项目范围。每次只问一个，等回答了再问下一个。

#### 问题1：为什么要做这个？
**解释：** 如果"为什么"说不清楚，后面怎么写都是空的。这是所有需求的根基。

#### 问题2：给谁用？
**解释：** 不接受"所有人"这种回答。要追问具体场景，比如"是程序员用还是产品经理用？是用在工作还是生活中？"

#### 问题3：怎么算成功？
**解释：** 叢能选一件事的话是什么？帮你砍需求，MVP要足够小。

#### 问题4：明确不做什么？
**解释：** 这个比"做什么"更重要——没有边界的PRD会失控。

#### 问题5：怎么算成功？
**解释：** 验收标准。没有验收标准的PRD就是许愿清单。

---

### 第三步：技术方案建议

根据用户的回答自动判断产品类型，授荐技术方案：

| 产品类型 | 推荐技术栈 | 为什么选这个 |
|---------|-----------|-------------|
| 纯展示类（导航站、主页） | HTML + Tailwind + 静态部署 | 最简单，不需要后端，半小时上线 |
| 有AI交互（内容生成、对话） | Next.js + AI API + Vercel | 前端后端一体化，AI集成方便 |
| 需要存数据（用户记录、历史） | Next.js + Supabase + Vercel | Supabase 免费额度够用，部署零配置 |
| 不确定 | Next.js + Tailwind + Vercel | 万能选择，社区最大，出了问题好找答案 |

---

### 第四步：生成一页纸PRD

把前面的信息整合，输出结构化文档：

```
# [产品名称]

## 产品定位
一句话说明这是什么产品。

## 目标用户
2-3句话描述谁会用，为什么需要。

## 核心功能（最多3个）

### 功能1
- 用户操作：xxx
- 预期结果：xxx
- 验收标准：xxx

### 功能2
- 用户操作：xxx
- 预期结果：xxx
- 验收标准：xxx

### 功能3
- 用户操作：xxx
- 预期结果：xxx
- 验收标准：xxx

## 明确不做
- 不做 xxx
- 不支持 xxx

## 技术方案
- 技术栈：xxx
- 选型理由：xxx
- 部署：xxx

## 产品风格
- UI风格：xxx
- 语言：中文/英文
- 色调：xxx

## 执行指令
把这份PRD直接给AI，告诉它：
1. 先搭项目骨架
2. 实现核心功能
3. 验证每个功能的验收标准
4. 部署上线

---
```

---

### 第五步：质量自检

生成后自动对照清单检查，每一项都要确认：

- [ ] 能否一句话说清楚这个产品是什么？
- [ ] MVP范围一个下午能做完吗？
- [ ] "明确不做"列了吗？
- [ ] 每个功能有验收标准吗？
- [ ] 技术方案匹配需求吗？
- [ ] 把这份PRD直接给AI，它能立刻开始工作吗？

如果任何一项不通过，主动提出修改建议。
